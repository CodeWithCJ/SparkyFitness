# Privacy Policy

Last updated: July 10, 2026

This policy explains how SparkyFitness handles information in the web and
mobile applications. SparkyFitness can be operated in two different ways, and
the party responsible for the server depends on the deployment you use.

## Deployment models

### Self-hosted deployments

When you connect to a self-hosted SparkyFitness server, that server's owner is
the data controller and administrator. Health, account, and fitness data are
sent only to the server you selected. Its operator controls hosting, backups,
retention, deletion, optional integrations, and any configured AI provider.

### Official hosted web deployment

The official hosted web service at `https://fitness.hala-apps.com` is operated
for SparkyFitness users using Vercel for application hosting and Supabase for
managed PostgreSQL. Unlike a purely local mobile installation, this service
must process and store account and health data to provide synchronization,
reports, and web access. Production secrets and database access are restricted
to the service operator and infrastructure providers needed to run the service.

## Information handled by SparkyFitness

Depending on the features you use, a SparkyFitness server may process:

- Account details such as name, email address, authentication records, active
  sessions, and security settings.
- Health and fitness information such as weight, measurements, steps, heart
  rate, blood oxygen, sleep, workouts, food diary entries, goals, medications,
  cycle or pregnancy information, and notes you enter.
- Optional images such as avatars, progress photos, exercise images, or
  nutrition labels when the deployment enables uploads.
- Integration credentials and OAuth tokens. Secrets and supported provider
  tokens are encrypted at rest with the server's configured encryption key.
- Operational request information made available to the server or hosting
  platform, such as timestamps, IP addresses, user agents, response status,
  and error logs used for security, reliability, and troubleshooting.

The mobile app may also store server connection information and session
material locally on the device so it can connect to the selected server.

## HUAWEI Health

HUAWEI Health linking is optional and requires explicit authorization through
Huawei's OAuth consent screen. The data flow is:

`Huawei wearable → HUAWEI Health phone app → Huawei cloud → SparkyFitness server`

SparkyFitness may request read access to steps, calories burned, distance,
heart-rate summaries, blood oxygen saturation, body weight, sleep records, and
workout/activity records. It uses authorized data only to create the user's
health history, diary entries, and reports inside SparkyFitness.

- Users can deny individual data permissions. Approved data continues to sync;
  unapproved data is skipped.
- SparkyFitness checks Huawei consent during every sync and honors later scope
  revocation or Huawei privacy-switch changes.
- Huawei access and refresh tokens are encrypted on the SparkyFitness server.
  They are strictly owner-only and are never exposed to family delegates.
- Selecting **Disconnect** asks Huawei to cancel SparkyFitness authorization
  and then removes locally stored tokens. If Huawei cannot confirm revocation,
  SparkyFitness leaves the local connection intact rather than displaying a
  misleading success state.
- If the server administrator has already removed the Huawei app credentials,
  the server cannot request remote revocation. It still deletes its local
  encrypted tokens and instructs the user to remove SparkyFitness in HUAWEI
  Health privacy settings.
- Disconnecting does not delete the user's original data from Huawei.
  Previously imported records remain on the SparkyFitness server until they
  are deleted there or removed under that server's retention policy.

Huawei processes the Huawei ID, consent, and health-cloud data under Huawei's
own terms and privacy notices. See the
[HUAWEI Health integration guide](/features/settings/huawei-health) for the
requested data types and connection controls.

## Camera and optional AI features

Barcode scanning is decoded on-device where supported. When a user explicitly
submits a nutrition-label image for AI analysis, the image is sent to the
selected SparkyFitness server and may be forwarded to the AI provider configured
by that server's administrator. The applicable AI provider then processes the
image under its own privacy terms. This feature is optional.

## How information is used

SparkyFitness uses information to:

- Authenticate users and protect accounts.
- Display diaries, check-ins, health history, goals, and reports.
- Synchronize data with services the user explicitly connects.
- Provide family-access features under the permissions chosen by the owner.
- Maintain security, diagnose failures, and operate the service.

SparkyFitness does not sell or rent personal or health data, serve targeted
advertising, or include behavioral advertising trackers.

## Sharing and service providers

Information is shared only as needed to provide a feature or operate the chosen
deployment. This can include:

- The self-hosted server operator selected by the user.
- Vercel and Supabase for the official hosted deployment.
- A connected health provider, such as Huawei, when the user initiates linking,
  synchronization, or revocation.
- An AI provider configured by the server administrator when the user invokes
  an optional AI feature.
- Authorities or other parties when required by applicable law or necessary to
  protect users, the service, or legal rights.

## Retention, export, and deletion

- On-device configuration and cached data remain until cleared by the user or
  removed with the app.
- Self-hosted server retention and backups are controlled by that server's
  operator.
- The official hosted deployment retains account and application data for as
  long as needed to provide the account and operate reliable backups, unless a
  shorter legal or operational requirement applies.
- Users can export the diary data currently supported by **Settings → Data
  Management**. Other records can be deleted through their normal application
  controls where available.
- To request deletion of an official hosted account or data that has no in-app
  deletion control, contact the address below from the account's registered
  email. Self-hosted users must contact their own server administrator.

Deleting a SparkyFitness copy does not automatically delete data retained by a
connected provider. Use that provider's controls for its original records.

## Security

SparkyFitness uses authenticated sessions, PostgreSQL Row-Level Security,
owner-only boundaries for sensitive integrations, encrypted supported secrets,
and HTTPS on the official hosted service. No system can guarantee absolute
security. Self-hosted operators are responsible for patching, HTTPS, database
security, backups, and access control on their own deployment.

## Children's privacy

The official hosted service is not intended for children under 13, and the
operator does not knowingly create hosted accounts for children under 13.
Family profiles must be managed by an authorized adult in accordance with
applicable law.
