# Saudi Arabic Product Terminology

_Working glossary for the SparkyFitness web, mobile, server, notifications, widgets, and Sparky AI experience._

## How to Use This Glossary

- Use the preferred term consistently unless the screen needs a shorter label.
- Use light Najdi wording in welcome, coaching, empty states, and casual prompts.
- Use professional Arabic in forms and precise Modern Standard Arabic for medical, privacy, security, and destructive-action copy.
- Translate the user's intent, not the English word order.
- Keep brand names, user content, URLs, API keys, medication names, and scientific unit symbols unchanged.

## Najdi Warmth: Approved Patterns

Use these sparingly and only where warmth improves the experience:

| Intent          | Preferred pattern                 |
| --------------- | --------------------------------- |
| Welcome         | `حياك في سباركي فتنس`             |
| Start           | `يلا نبدأ`                        |
| Completed goal  | `كفو، كملت هدفك اليوم.`           |
| Empty day       | `ما سجّلت شيء للحين.`             |
| Ask what to add | `وش ودّك تسجّل؟`                  |
| Gentle recovery | `ما ضبطت هالمرة. حاول مرة ثانية.` |

Do not repeat these phrases across every card or toast. Most interface labels should stay quiet and direct.

## Error and Confirmation Patterns

- Prefer cause + next action: `تعذر حفظ الوجبة. تأكد من اتصالك وحاول مرة ثانية.`
- Name destructive scope: `حذف هذا التمرين من السجل؟ ما تقدر تتراجع بعد الحذف.`
- Do not blame the user: avoid `أدخلت بيانات خاطئة`; use `تأكد من البيانات المدخلة`.
- Do not promise outcomes: avoid `بيتم الحفظ أكيد`; report actual state.
- Do not expose status codes, translation keys, or raw server messages when a user-safe localized message exists.
